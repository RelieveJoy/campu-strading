### 4. 缓存策略优化

**当前问题**：首页缓存固定 5 分钟 TTL，发布商品后要等 5 分钟才能看到。

**计划**：
- [ ] **缓存预热**：`@PostConstruct` 启动时加载首页数据到 Redis
- [ ] **主动失效**：发布/编辑/下架商品时立即删除首页缓存（`redisTemplate.delete(key)`）
- [ ] **缓存雪崩防护**：TTL 加随机偏移（`5 * 60 + RandomUtils.nextInt(0, 60)`）
- [ ] **缓存穿透防护**：空结果也缓存（短 TTL）
- [ ] 考虑使用 `@Cacheable` / `@CacheEvict` 注解替代手动操作 Redis

---

### 5. 消息队列通知

**当前问题**：下单后卖家无感知，只能靠轮询。

**计划**（选择 RabbitMQ 或 RocketMQ）：
- [ ] Docker Compose 增加 MQ 容器
- [ ] 引入 `spring-boot-starter-amqp`（RabbitMQ）
- [ ] **下单通知**：买家下单 → MQ → 卖家收到通知
- [ ] **订单状态变更通知**：确认/取消 → MQ → 买家收到通知
- [ ] 消息持久化 + 手动 ACK，保证不丢消息
- [ ] 前端通知中心（右上角消息图标 + 未读红点）
- [ ] 可选：通知落库（`notification` 表）做历史记录

---

### 6. 分布式锁

**当前问题**：`ViewCountSyncTask` 定时任务多实例部署时会重复同步。

**计划**：
- [ ] 引入 **Redisson**（`redisson-spring-boot-starter`）
- [ ] `ViewCountSyncTask` 加 `@SchedulerLock` 或 Redisson 分布式锁
- [ ] 下单防超卖：从 SQL CAS 改为 Redisson 分布式锁（更通用，面试更有的聊）
- [ ] 封装 `@DistributedLock` 注解（AOP 实现），方便其他地方复用

### 7. WebSocket 买家卖家聊天

**当前问题**：买卖双方只能看到对方手机号，没有站内沟通渠道。

**计划**：
- [ ] 引入 `spring-boot-starter-websocket`
- [ ] 建立 WebSocket 连接：基于订单 ID 创建会话
- [ ] 消息持久化：`message` 表（sender_id、receiver_id、order_id、content、is_read、send_time）
- [ ] 心跳保活 + 断线重连
- [ ] 前端聊天窗口（订单详情页底部 / 独立聊天页）
- [ ] 未读消息红点
- [ ] 可选：敏感词过滤

### 8. 搜索升级

**当前问题**：MySQL `LIKE %keyword%`，数据量大时走不了索引。

**计划**：

**轻量方案（先做）**：
- [ ] MySQL 全文索引（`FULLTEXT INDEX`）+ `MATCH ... AGAINST`

**重量方案（面试加分项）**：
- [ ] Docker Compose 增加 Elasticsearch 容器
- [ ] `spring-boot-starter-data-elasticsearch`
- [ ] 商品发布/编辑时同步到 ES 索引
- [ ] 搜索接口改为从 ES 查询（高亮 + 分词 + 权重排序）
- [ ] 可选：IK 中文分词器

### 11. 接口限流

**当前问题**：登录/注册接口无防护，有被暴力攻击的风险。

**计划**：
- [ ] 自定义 `@RateLimiter` 注解（基于 IP + Redis 滑动窗口）
- [ ] AOP 实现：超出限制返回 429 Too Many Requests
- [ ] 对 `/api/users/login`、`/api/users/register`、`/api/common/upload` 加限流
- [ ] 配置不同接口的限流阈值（登录 10次/分钟、上传 20次/分钟 等）
- [ ] 或直接使用 `bucket4j-spring-boot-starter`
