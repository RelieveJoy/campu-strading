## 二、各模块详细说明

### 1. 移除硬编码密钥

**问题**：`application-dev.yml` 直接包含阿里云 OSS Key/Secret、微信 AppID/Secret、数据库密码。

**改动**：
- `application-dev.yml`：所有密钥替换为 `${环境变量}` 占位符
- `.env`：存放真实值（被 `.gitignore` 排除）
- `.gitignore`：移除 `application-dev.yml` 排除规则，允许推送占位符版本

**涉及文件**：`application-dev.yml`、`.env`、`.gitignore`

### 2. 服务端参数校验

- `ItemPageQueryDTO`、`OrdersPageQueryDTO` 分页参数加 `@Min`、`@Max` 限制

**涉及文件**：
- `UserLoginDTO`、`UserRegisterDTO`、`UserUpdateDTO`、`PasswordEditDTO`
- `ItemDTO`、`OrdersDTO`、`FavoriteDTO`
- `ItemPageQueryDTO`、`OrdersPageQueryDTO`

### 4. 缓存策略优化

**已实现**：

| 能力 | 实现方式 |
|------|---------|
| **缓存预热** | `CacheWarmupRunner` 启动时调用 `itemService.getHomeItems()` 加载到 Redis |
| **主动失效** | `@CacheInvalidate` 注解 + `CacheInvalidateAspect` 切面，增删改后立即删除缓存 |
| **缓存雪崩防护** | 首页缓存 TTL 加随机偏移（`5 + ThreadLocalRandom.nextLong(1)` 分钟） |
| **缓存穿透防护** | 空结果也缓存，TTL 较短（1~2 分钟） |

**涉及文件**：`ItemServiceImpl`（getHomeItems 方法）、`CacheWarmupRunner`（预热）、`CacheInvalidateAspect`（主动失效）

### 5. 消息队列通知

**架构**：

```
下单/确认/取消 → RabbitMQ → NotificationConsumer
  → ① 入 notification 表（离线用户上线后可见）
  → ② WebSocket 推 /topic/notify.{userId}（在线用户即时收到）
```

**消息路由**：
- 交换机：`campus.trading.exchange`（DirectExchange）
- 队列：`campus.trading.order.notify`（durable=true，持久化）
- 路由键：`order.event`

**新增依赖**：`spring-boot-starter-amqp`

**新增/修改文件**：
- `dto/OrderNotificationDTO`（消息体：orderId、itemTitle、amount、buyerId/SellerId、triggerUserId、type）
- `config/RabbitMqConfig`（队列/交换机/绑定声明 + JSON 序列化）
- `mq/NotificationProducer`（发消息）
- `mq/NotificationConsumer`（收消息 → 入库 + WebSocket 推送）
- `OrderServiceImpl`（下单/确认/取消后调用 `sendNotification`）
- `docker-compose.yml`（RabbitMQ 容器 + 管理面板 15672）

**通知持久化**：
- `notification` 表（user_id、initiator_id、initiator_name、type、order_id、item_title、content、is_read）
- `NotificationController`：`GET /api/notifications`（查列表）、`POST /api/notifications/read`（全部已读）
- 前端：`MyMessages.vue` 打开通知列表时自动调用已读接口

**通知内容**（区分买卖双方）：

| 事件 | 买家看到 | 卖家看到 |
|------|---------|---------|
| 下单 | 你已下单「XX」¥XX，等待卖家确认 | 买家 XX 购买了你的「XX」¥XX，请尽快处理 |
| 确认 | 卖家已确认完成订单「XX」 | 订单「XX」已确认完成 |
| 取消 | 你已取消订单「XX」 | XX 取消了订单「XX」 |

### 6. 分布式锁

**实现**：

- 引入 `redisson-spring-boot-starter:3.34.0`
- 自定义 `@DistributedLock` 注解（支持 key、waitTime、leaseTime、timeUnit）
- `DistributedLockAspect` 切面（`@Around` 拦截，tryLock + unlock）
- `@ConditionalOnBean(RedissonClient.class)` 确保 Redisson 不可用时切面不生效
- `ViewCountSyncTask.syncViewCount()` 加 `@DistributedLock(key = "viewCountSync", ...)`

**配置**：`application.yml` 中 `spring.redis.redisson.config` 内联 YAML（`singleServerConfig`）

**适用场景**：多实例部署时防止多步操作并发冲突。当前 `getAndSet` 是单原子命令不需要锁，但基础设施已就位供未来扩展。

**新增文件**：`annotation/DistributedLock`、`aspect/DistributedLockAspect`

### 7. WebSocket 买家卖家聊天

**架构**：STOMP over WebSocket + SockJS 降级

```
发消息：POST /api/messages { itemId, receiverId, content }
  → MessageService.send() 落库
  → WebSocket 推 /topic/chat.{senderId}.{itemId}
  → WebSocket 推 /topic/chat.{receiverId}.{itemId}

查历史：GET /api/messages/{itemId}?receiverId=xxx
  → 只返回两方的消息（不串线）

对话列表：GET /api/messages/conversations
  → 按 (商品, 对方) 分组，含未读数、最后消息、对方信息
```

**WebSocket 端点**：`ws://host:8080/ws?token=xxx`

**防串线**：按用户 + 商品组合频道（`/topic/chat.{userId}.{itemId}`），非收发双方收不到消息

**新增文件**：
- `entity/Message`（item_id、sender_id、receiver_id、content、is_read）
- `dto/MessageDTO`、`vo/MessageVO`、`vo/ConversationVO`
- `mapper/MessageMapper` + `mapper/MessageMapper.xml`
- `service/MessageService` + `impl/MessageServiceImpl`
- `config/WebSocketConfig`（STOMP 端点 + 消息代理）
- `interceptor/WebSocketAuthInterceptor`（握手时校验 JWT）
- `controller/ChatController`（REST 发消息/查历史 + WebSocket 推送）
- `db/init.sql` 加 `message` 表

**前端**：
- `components/ChatModal.vue`（聊天弹窗）
- `api/chat.js`（`getMessages`、`sendMessage`、`getConversations`、`markRead`）
- `views/ItemDetail.vue`（"联系卖家"按钮 → 打开 ChatModal）
- `views/MyMessages.vue`（对话列表 + 通知列表）

### 8. 搜索升级

**双引擎架构**：

```
有关键词 → Elasticsearch（IK 分词 + .char 按字兜底）
  → ES 返回 itemId 列表 → MySQL pageQueryByIds 补全数据
  
无关键词 → MySQL MATCH AGAINST（FULLTEXT 索引）
```

**MySQL 全文索引**：
- `item` 表 `FULLTEXT KEY ft_title_desc (title, description)`
- `pageQuery` SQL 改为 `MATCH ... AGAINST IN NATURAL LANGUAGE MODE`

**Elasticsearch**：
- Docker Compose 加 `elasticsearch:7.17.25` + IK 分词器 `analysis-ik`
- `@Document(indexName = "campus_items")`：`ItemDocument`（按商品字段建模）
- `title` 字段三分身：主字段 IK 分词、`.keyword` 原始文本、`.char` standard 按字分词
- 搜索策略：`should` 组合——IK `multi_match` + `title.char` 按字 AND 匹配
- 全量同步：`ItemSearchService.indexAll()` 启动时自动执行
- 增量同步：`ItemService.add/update/delete/relist` 自动调 `index()`
- ES 不可用时自动回退 MySQL

**动态排序**：支持 `newest`（最新）、`hottest`（最热）、`price_asc`（价格↑）、`price_desc`（价格↓）

**新增/修改文件**：
- `docker-compose.yml`（ES 容器，端口 9200）
- `pom.xml`（`spring-boot-starter-data-elasticsearch`）
- `application.yml`（`spring.elasticsearch.uris`）
- `es/ItemDocument`（ES 文档模型，IK + keyword + char 多字段）
- `es/ItemSearchRepository`（Spring Data ES Repository）
- `es/ItemSearchService`（搜索 + 同步逻辑）
- `ItemMapper` + `ItemMapper.xml`（`pageQueryByIds` + 动态排序 + FULLTEXT）
- `ItemPageQueryDTO`（加 `sort`、`itemCondition` 字段）
- `ItemServiceImpl`（ES 同步钩子）
- `OrderServiceImpl`（下单/取消时同步 ES）
- `CacheWarmupRunner`（启动时 ES 全量同步）
