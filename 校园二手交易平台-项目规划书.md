# 校园二手交易平台 — 项目规划书

---

## 一、项目概述

### 1.1 项目名称
**校园二手交易平台** — Campus Second-Hand Trading Platform

### 1.2 项目背景
高校校园内每年毕业季、开学季存在大量闲置物品流转需求，学生之间缺乏统一、可信的线上交易渠道。本项目旨在为校内学生提供一个便捷的二手物品发布、浏览、交易的 Web 平台。

### 1.3 项目目标
搭建一个前后端分离的 Web 应用，实现二手商品发布、浏览搜索、下单交易、收藏管理等核心功能，满足课程大作业基本要求与进阶要求。

## 二、需求分析

### 2.1 用户角色

| 角色 | 描述 |
|------|------|
| 普通用户（学生） | 注册登录、浏览商品、搜索商品、发布商品、购买商品、管理订单、收藏商品 |

> 注：本项目仅设置单一用户角色，不区分管理员，简化开发。每个用户既是买家也是卖家。

### 2.2 功能模块

#### 模块一：用户模块
| 编号 | 功能 | 描述 |
|------|------|------|
| U-01 | 用户注册（✅） | 填写学号、姓名、手机号、密码等信息注册账号 |
| U-02 | 用户登录（✅） | 使用学号+密码登录（悬浮弹窗，背景虚化） |
| U-03 | 个人信息管理（✅） | 查看和修改个人资料 |
| U-04 | 密码修改（✅） | 修改登录密码 |
| U-05 | 用户登出（✅） | 登出账号，Token 加入 Redis 黑名单 |

#### 模块二：商品模块（核心）
| 编号 | 功能 | 描述 |
|------|------|------|
| I-01 | 发布商品（✅） | 填写标题、描述、价格、原价、分类、上传图片至阿里云 OSS 获得 URL，发布二手商品 |
| I-02 | 编辑商品（✅） | 修改已发布但未售出的商品信息 |
| I-03 | 下架商品（✅） | 下架已发布商品 |
| I-04 | 重新上架（✅） | 将已下架商品恢复在售状态 |
| I-05 | 全量查询（✅） | 首页分页展示所有在售商品列表 |
| I-06 | 模糊条件查询（✅） | 按商品名称模糊搜索 + 按分类筛选 + 按价格区间筛选 |
| I-07 | 商品详情（✅） | 查看商品详细信息、卖家信息 |

#### 模块三：交易模块
| 编号 | 功能 | 描述 |
|------|------|------|
| O-01 | 创建订单（✅） | 买家对商品下单购买，商品自动变为已售出（@Transactional） |
| O-02 | 我的购买（✅） | 查看我作为买家的所有订单（tab 切换） |
| O-03 | 我的售出（✅） | 查看我作为卖家的所有订单（tab 切换） |
| O-04 | 订单状态流转（✅） | 待确认 → 已完成 / 已取消（取消后商品恢复在售） |

#### 模块四：收藏模块（进阶）
| 编号 | 功能 | 描述 |
|------|------|------|
| F-01 | 收藏商品（✅） | 收藏感兴趣的商品 |
| F-02 | 取消收藏（✅） | 取消已收藏的商品 |
| F-03 | 我的收藏（✅） | 查看收藏列表 |

#### 模块六：Redis 应用
| 编号 | 功能 | 描述 |
|------|------|------|
| R-01 | JWT 登出黑名单 | 用户登出时将 Token 存入 Redis，设置过期时间 = Token 剩余有效期，拦截器每次校验时先查黑名单 |
| R-02 | 首页商品缓存 | 首页热门/最新商品列表缓存在 Redis，TTL 5 分钟，减少数据库查询压力 |
| R-03 | 商品浏览量计数 | 每次查看商品详情时 Redis INCR 浏览量，每 10 次访问异步回写 MySQL，避免频繁写库 |

#### 模块五：数据可视化（进阶）
| 编号 | 功能 | 描述 |
|------|------|------|
| V-01 | 分类商品数量统计（✅） | ECharts 饼图展示各分类商品占比 |
| V-02 | 价格区间分布（✅） | ECharts 柱状图展示商品价格区间分布 |
| V-03 | 近期交易趋势（✅） | ECharts 折线图展示近30天交易量趋势 |

### 2.3 功能关联关系

```
用户模块 ←→ 商品模块（用户发布商品）
用户模块 ←→ 交易模块（用户买卖）
用户模块 ←→ 收藏模块（用户收藏商品）
商品模块 ←→ 交易模块（商品被下单后状态变更）
商品模块 ←→ 收藏模块（商品被收藏）
交易模块 ←→ 数据可视化（交易数据统计图表来源）
```

### 4.2 表结构设计

#### user（用户表）— 8 个字段
```sql
CREATE TABLE user (
    user_id      BIGINT        PRIMARY KEY AUTO_INCREMENT  COMMENT '用户ID',
    student_id   VARCHAR(20)   NOT NULL UNIQUE             COMMENT '学号（登录账号）',
    username     VARCHAR(50)   NOT NULL                    COMMENT '用户姓名',
    password     VARCHAR(255)  NOT NULL                    COMMENT '登录密码（BCrypt加密存储）',
    phone        VARCHAR(20)                                COMMENT '手机号',
    avatar       VARCHAR(500)                               COMMENT '头像URL',
    status       TINYINT       DEFAULT 1                   COMMENT '状态：1启用 0禁用',
    create_time  DATETIME      DEFAULT CURRENT_TIMESTAMP   COMMENT '注册时间',
    update_time  DATETIME      DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    create_user  BIGINT                                     COMMENT '创建人ID',
    update_user  BIGINT                                     COMMENT '修改人ID'
) COMMENT '用户表';
```

#### item（商品表）— 11 个字段
```sql
CREATE TABLE item (
    item_id        BIGINT        PRIMARY KEY AUTO_INCREMENT  COMMENT '商品ID',
    seller_id      BIGINT        NOT NULL                    COMMENT '卖家用户ID',
    title          VARCHAR(100)  NOT NULL                    COMMENT '商品标题',
    description    TEXT                                      COMMENT '商品描述',
    price          DECIMAL(10,2) NOT NULL                    COMMENT '售价',
    original_price DECIMAL(10,2)                             COMMENT '原价',
    category_id    BIGINT        NOT NULL                    COMMENT '分类ID（关联category表）',
    status         TINYINT       DEFAULT 1                   COMMENT '状态：1在售 0下架 2已售出',
    image_url      VARCHAR(500)                              COMMENT '商品图片URL',
    view_count     INT           DEFAULT 0                   COMMENT '浏览次数',
    create_time    DATETIME      DEFAULT CURRENT_TIMESTAMP   COMMENT '发布时间',
    update_time    DATETIME      DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    create_user    BIGINT                                     COMMENT '创建人ID',
    update_user    BIGINT                                     COMMENT '修改人ID',
    FOREIGN KEY (seller_id) REFERENCES user(user_id)
) COMMENT '商品表';
```

#### category（分类表）— 4 个字段
```sql
CREATE TABLE category (
    category_id   BIGINT       PRIMARY KEY AUTO_INCREMENT  COMMENT '分类ID',
    name          VARCHAR(30)  NOT NULL                    COMMENT '分类名称',
    sort          INT          DEFAULT 0                   COMMENT '排序',
    status        TINYINT      DEFAULT 1                   COMMENT '状态：1启用 0禁用',
    create_time   DATETIME     DEFAULT CURRENT_TIMESTAMP   COMMENT '创建时间',
    update_time   DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    create_user   BIGINT                                    COMMENT '创建人ID',
    update_user   BIGINT                                    COMMENT '修改人ID'
) COMMENT '分类表';
```

#### orders（订单表）— 8 个字段
```sql
CREATE TABLE orders (
    order_id    BIGINT        PRIMARY KEY AUTO_INCREMENT  COMMENT '订单ID',
    item_id     BIGINT        NOT NULL                    COMMENT '商品ID',
    buyer_id    BIGINT        NOT NULL                    COMMENT '买家用户ID',
    seller_id   BIGINT        NOT NULL                    COMMENT '卖家用户ID',
    amount      DECIMAL(10,2) NOT NULL                    COMMENT '成交金额',
    status      TINYINT       DEFAULT 1                   COMMENT '状态：1待确认 2已完成 3已取消',
    create_time DATETIME      DEFAULT CURRENT_TIMESTAMP   COMMENT '下单时间',
    update_time DATETIME      DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    create_user  BIGINT                                     COMMENT '创建人ID',
    update_user  BIGINT                                     COMMENT '修改人ID',
    FOREIGN KEY (item_id)   REFERENCES item(item_id),
    FOREIGN KEY (buyer_id)  REFERENCES user(user_id),
    FOREIGN KEY (seller_id) REFERENCES user(user_id)
) COMMENT '订单表';
```

#### favorite（收藏表）— 3 个字段
```sql
CREATE TABLE favorite (
    favorite_id BIGINT   PRIMARY KEY AUTO_INCREMENT  COMMENT '收藏ID',
    user_id     BIGINT   NOT NULL                    COMMENT '用户ID',
    item_id     BIGINT   NOT NULL                    COMMENT '商品ID',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP   COMMENT '收藏时间',
    FOREIGN KEY (user_id) REFERENCES user(user_id),
    FOREIGN KEY (item_id) REFERENCES item(item_id),
    UNIQUE KEY uk_user_item (user_id, item_id)       COMMENT '同一用户不能重复收藏同一商品'
) COMMENT '收藏表';
```

### 4.3 公共字段说明
每张业务表均包含以下公共字段，通过 AOP 自动填充，无需手动设置：
- `create_time` — 创建时间，INSERT 时自动填充当前时间
- `update_time` — 修改时间，INSERT/UPDATE 时自动填充当前时间
- `create_user` — 创建人ID，INSERT 时自动从 BaseContext 获取当前登录用户ID
- `update_user` — 修改人ID，INSERT/UPDATE 时自动从 BaseContext 获取当前登录用户ID

### 4.4 Entity / DTO / VO 代码规范

**Entity（数据库实体）**— 使用 `@Builder` 构建模式，与表结构一一对应：

```java
@Data
@Builder
@NoArgsConstructor
@AllArgsConstructor
public class Item implements Serializable {
    private static final long serialVersionUID = 1L;

    private Long itemId;
    private Long sellerId;
    private String title;
    private String description;
    private BigDecimal price;
    private BigDecimal originalPrice;
    private Long categoryId;
    private Integer status;
    private String imageUrl;
    private Integer viewCount;
    private LocalDateTime createTime;
    private LocalDateTime updateTime;
    private Long createUser;
    private Long updateUser;
}
```

**DTO（数据传输对象）**— 接收前端请求体，简洁无冗余注解：

```java
@Data
@ApiModel(description = "商品分页查询条件")
public class ItemPageQueryDTO implements Serializable {
    @ApiModelProperty("页码")
    private Integer page;
    @ApiModelProperty("每页条数")
    private Integer pageSize;
    @ApiModelProperty("商品名称关键词")
    private String keyword;
    @ApiModelProperty("分类ID")
    private Long categoryId;
    @ApiModelProperty("状态")
    private Integer status;
}
```

**VO（视图对象）**— 返回给前端的数据，可跨表组合：

```java
@Data
@Builder
@NoArgsConstructor
@AllArgsConstructor
@ApiModel(description = "商品详情（含卖家信息）")
public class ItemDetailVO implements Serializable {
    @ApiModelProperty("商品ID")
    private Long itemId;
    @ApiModelProperty("商品标题")
    private String title;
    // ... 其它商品字段
    @ApiModelProperty("卖家姓名")
    private String sellerName;
    @ApiModelProperty("分类名称")
    private String categoryName;
}
```

### 4.5 多表连接查询示例
- **商品详情查询**：`item LEFT JOIN user ON item.seller_id = user.user_id`（商品 + 卖家信息）
- **我的订单查询**：`orders JOIN item ON orders.item_id = item.item_id JOIN user ON orders.buyer_id = user.user_id`（订单 + 商品 + 买家信息）
- **首页商品列表**：`item LEFT JOIN user ON item.seller_id = user.user_id LEFT JOIN category ON item.category_id = category.category_id`（商品 + 卖家姓名 + 分类名称）

### 7A.3 ThreadLocal 上下文（`context/`）

通过 `BaseContext` 基于 ThreadLocal 存储当前请求的用户ID，同一线程内任意位置均可获取，避免层层传参：

```java
// BaseContext.java
public class BaseContext {
    public static ThreadLocal<Long> threadLocal = new ThreadLocal<>();

    public static void setCurrentId(Long id) { threadLocal.set(id); }
    public static Long getCurrentId() { return threadLocal.get(); }
    public static void removeCurrentId() { threadLocal.remove(); }
}
```

**使用场景**：
- JWT 拦截器校验通过后 `BaseContext.setCurrentId(userId)` 存储当前用户ID
- Service 层通过 `BaseContext.getCurrentId()` 获取当前操作用户
- 请求结束后由拦截器的 `afterCompletion` 调用 `removeCurrentId()` 清理，防止内存泄漏

### 7A.4 AOP 公共字段自动填充（`annotation/` + `aspect/`）

通过自定义注解 `@AutoFill` + AspectJ 切面，自动填充 `create_time`/`update_time`/`create_user`/`update_user` 四个公共字段，无需在每个 Service 方法中手动设置。

**自定义注解**：
```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface AutoFill {
    OperationType value();  // INSERT 或 UPDATE
}
```

**切面实现**：
```java
@Aspect
@Component
@Slf4j
public class AutoFillAspect {
    @Pointcut("execution(* com.campus.mapper.*.*(..)) && @annotation(com.campus.annotation.AutoFill)")
    public void autoFillPointCut() {}

    @Before("autoFillPointCut()")
    public void autoFill(JoinPoint joinPoint) {
        // 获取方法上的 @AutoFill 注解，判断是 INSERT 还是 UPDATE
        // INSERT: 反射调用 setCreateTime/setUpdateTime/setCreateUser/setUpdateUser
        // UPDATE: 反射调用 setUpdateTime/setUpdateUser
        // 时间取 LocalDateTime.now()，用户ID取 BaseContext.getCurrentId()
    }
}
```

**使用方式** — 在 Mapper 方法上标注：
```java
@AutoFill(value = OperationType.INSERT)
void insert(Item item);

@AutoFill(value = OperationType.UPDATE)
void update(Item item);
```

### 7A.5 JWT 令牌校验拦截器（`interceptor/`）

```java
@Component
@Slf4j
public class JwtTokenInterceptor implements HandlerInterceptor {

    @Autowired
    private JwtProperties jwtProperties;

    public boolean preHandle(HttpServletRequest request,
                             HttpServletResponse response,
                             Object handler) throws Exception {
        // 1. 如果不是 Controller 方法，直接放行
        if (!(handler instanceof HandlerMethod)) return true;

        // 2. 从请求头获取 Token
        String token = request.getHeader(jwtProperties.getUserTokenName());

        // 2. 检查 Token 是否在黑名单中（已登出）
        Boolean isBlacklisted = redisTemplate.hasKey("blacklist:token:" + token);
        if (Boolean.TRUE.equals(isBlacklisted)) {
            response.setStatus(401);
            return false;
        }

        // 3. 解析 Token，校验有效性
        try {
            Claims claims = JwtUtil.parseJWT(jwtProperties.getUserSecretKey(), token);
            Long userId = Long.valueOf(claims.get(JwtClaimsConstant.USER_ID).toString());
            BaseContext.setCurrentId(userId);  // 存入 ThreadLocal
            return true;  // 放行
        } catch (Exception ex) {
            response.setStatus(401);  // 校验失败，返回 401
            return false;
        }
    }

    // afterCompletion 中调用 BaseContext.removeCurrentId() 清理
}
```

**注册拦截器**（`WebMvcConfiguration.java`）：

```java
        RedisTemplate redisTemplate = new RedisTemplate();
        redisTemplate.setConnectionFactory(factory);
        redisTemplate.setKeySerializer(new StringRedisSerializer());
        return redisTemplate;
    }
}
```

Redis 三项应用：
1. **JWT 登出黑名单** — Key: `blacklist:token:{token}`，Value: `1`，TTL = Token 剩余有效期
2. **首页商品缓存** — Key: `cache:home:items`，TTL = 5 分钟，减少数据库压力
3. **商品浏览量** — Key: `view_count:{itemId}`，每次查看 INCR，每 10 次异步回写 MySQL

## 十、风险与注意事项

1. **多模块 Maven 依赖管理**：父 POM 通过 `<dependencyManagement>` 统一管理版本号，子模块无需指定版本。`campus-server` 依赖 `campus-common` 和 `campus-pojo`，注意模块间依赖关系。
2. **图片上传**：前端使用阿里云 OSS SDK 直传图片，拿到 URL 后提交给后端存储。需提前在阿里云控制台创建 Bucket 并配置跨域规则。后端通过 `AliOssUtil` 工具类封装 OSS 操作。
3. **登录态管理**：JWT Token 登录成功后将 Token 存入 localStorage，每次请求通过 Axios 拦截器自动携带 Token 请求头（请求头名称由 `jwtProperties.getUserTokenName()` 配置）。后端 `JwtTokenInterceptor` 从请求头解析 Token，校验通过后将 userId 存入 `BaseContext`。
4. **跨域处理**：使用 Nginx 反向代理统一入口，前端请求 `/api/*` 全部代理到后端 8080 端口，从浏览器角度看是同源请求，天然避免跨域问题，无需在 Spring Boot 中配置 CORS。
5. **数据库事务**：下单等涉及多表写操作的业务使用 `@Transactional` 注解保证事务原子性。Spring Boot 启动类需添加 `@EnableTransactionManagement`。
6. **密码安全**：使用 `BCryptPasswordEncoder` 对密码做 BCrypt 哈希（自动加盐）后存储，登录时通过 `BCryptPasswordEncoder.matches()` 比对。实体返回前端时密码字段脱敏为 `"****"`。
7. **公共字段自动填充**：各表的 `create_time`/`update_time`/`create_user`/`update_user` 通过 AOP 切面自动填充，开发时只需在 Mapper 方法上标注 `@AutoFill(OperationType.INSERT/UPDATE)` 注解，无需手动调用 setter。
8. **异常处理**：Service 层遇到业务异常直接 `throw new XxxException(MessageConstant.XXX)` 抛出，由 `GlobalExceptionHandler` 统一捕获并返回 `Result.error(msg)`，Controller 层无需 try-catch。
9. **Redis 集成**：通过 `RedisConfiguration` 配置 RedisTemplate 序列化规则。Token 黑名单 Key 设计为 `blacklist:token:{token}`，TTL = Token 剩余有效期。首页缓存 Key 设计为 `cache:home:items`，TTL = 5 分钟。浏览量 Key 设计为 `view_count:{itemId}`。
10. **接口文档**：引入 Knife4j 自动生成 Swagger 接口文档，Controller 和方法上使用 `@Api` / `@ApiOperation` 注解标注，访问 `/doc.html` 可在线调试。
11. **Docker 部署注意**：确保 Docker Desktop 已安装。MySQL 初始化 SQL 通过 `docker-entrypoint-initdb.d` 挂载自动建表。后端容器通过容器名（mysql、redis）访问数据库和缓存服务，而非 `localhost`。
12. **Knife4j 与静态资源映射**：Spring Boot 2.7.x 中 Knife4j 需在 `WebMvcConfiguration` 中手动添加 `/doc.html` 和 `/webjars/**` 的静态资源映射，否则文档页无法访问。
